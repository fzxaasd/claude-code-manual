# Companion/Buddy System

> Based on source code `src/buddy/` complete analysis

---

## Overview

Companion (Buddy) is a desktop companion system that interacts with users through ASCII sprites and speech bubbles. It is not an Agent, but a standalone UI component system.

**Note**: The `/buddy pet` command **does not exist**. The pet animation is triggered by the `companionPetAt` timestamp in AppState.

---

| File | Component | Description |

## Generation Mechanism

### Mulberry32 PRNG

Uses the Mulberry32 pseudorandom number generator to ensure determinism:

```typescript
function mulberry32(seed: number): () => number
```

### Hash Generation

```typescript
function hashString(s: string): number
```

Uses `userId` as the seed to ensure each user generates the same companion.

### roll() Function

```typescript
export function roll(userId: string): Roll
```

Where the `Roll` type is defined as:

```typescript
type Roll = {
  bones: CompanionBones
  inspirationSeed: number
}
```

Process:
1. Generate seed using `hashString(userId)`
2. Determine rarity via PRNG
3. Assign species, eyes, and hat
4. Roll for shiny status
5. Assign five stats

## CompanionSoul

The companion's soul is generated by the model (non-deterministic):

```typescript
type CompanionSoul = {
  name: string        // companion name
  personality: string // personality description
}
```

The companion's name and personality are stored in the config after **hatching** and will not change again.

## CompanionObserver

Companion reaction generation system:

```typescript
fireCompanionObserver(messages: Message[], onReaction: (reaction: string) => void): Promise<void>
```

Uses LLM to generate the companion's reaction to user input. This function is called in `REPL.tsx`, where the first argument is the current message array and the second argument is a callback used to update `AppState.companionReaction`.
